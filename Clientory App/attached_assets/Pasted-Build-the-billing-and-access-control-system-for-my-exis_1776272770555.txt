Build the billing and access-control system for my existing SaaS product, Clientory.

IMPORTANT: the core product is already built. Do not rebuild the app from scratch. First inspect the current codebase and existing architecture, then integrate billing into the current product with the least disruptive approach.

## Product context
Clientory helps users measure AI visibility for businesses.
The product already does the core workflow:
- collect business information
- generate prompts
- run prompts through multiple LLMs
- return a final AI visibility report

Now I want to tie payment directly to the product so usage is gated properly.

## Core business rules
Please implement billing based on these exact rules:

### What counts as 1 scan
1 scan = 1 full report run for 1 business/account, even if that internally hits many prompts and models.

### Free usage
- A user gets 1 free report total
- After the free report is used, the user can still view that old report
- But the user cannot run new scans or rerun reports unless they pay

### Paid model
Use subscription billing first.

Pricing:
- $5/month per business
- Each paid business includes 4 scans per billing cycle
- The product language can describe this as “up to 1 scan per week” but enforce it as 4 scans per billing cycle for now
- Do not implement weekly scheduling logic right now
- Users may run scans ad hoc
- Structure the code so scheduled scans could be added later, but do not build monthly monitoring or automation now

### Billing entity
- Billing is tied to the user for now
- But subscriptions are purchased per business profile
- One user can have multiple businesses
- Each business should have its own subscription state and scan allowance
- Example: if a user has 3 businesses, they can pay $5/month for each business separately

### Cancellation / failed payment behavior
- If a subscription is canceled, keep paid access until the end of the billing cycle
- If payment fails, also keep access until the end of the current billing cycle
- After the billing period ends without an active paid subscription, block new scans and reruns
- Prior reports should remain viewable in read-only mode

### No trial beyond free report
- No free trial
- No tax handling for now
- No invoice customization needed for now

## Value sold
The product is selling AI visibility monitoring, not raw API usage.
The UX and copy should reflect that.

## Feature entitlements
Please enforce these feature rules:

### Free user
- Can submit email and verify it
- Can create/use the product enough to get 1 free report
- Can view that free report later
- Cannot rerun scans after the free report is used
- Cannot create additional paid-only business tracking flows unless unlocked
- Does not get paid exports/features

### Paid business subscription
For each paid business subscription, allow:
- up to 4 scans per billing cycle
- report history
- export PDF/CSV
- recommendations
- saved business profiles

When the business reaches 4 scans in the cycle:
- block new scans
- show a clear limit message
- do not run any LLM jobs once blocked

## Product/UX requirements
Build a strong MVP billing UX with:

1. Pricing page
2. Settings / Billing page
3. Upgrade modal when a user is blocked from scanning

### Pricing page
Should clearly explain:
- 1 free report
- Then $5/month per business
- 4 scans per billing cycle
- includes report history, exports, recommendations, and saved profiles

### Billing/settings page
Show:
- current businesses
- whether each business is free or paid
- subscription status per business
- scans used this billing cycle per business
- scans remaining per business
- renewal/reset date
- manage billing button
- subscribe button for unpaid businesses

### Upgrade modal
When a blocked user tries to run a scan:
- explain why they are blocked
- explain whether they used their free report or hit their plan limit
- offer upgrade path for that specific business

Use clear product copy like:
- “You’ve already used your free report.”
- “Subscribe to continue monitoring this business in AI search results.”
- “You’ve used 4 of 4 scans for this billing cycle.”
- “Your scan allowance resets on [date].”
